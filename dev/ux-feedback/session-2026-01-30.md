# UX Feedback Session - 2026-01-30

## Summary
Played full game as Runner vs heuristic Corp. **Runner won 6-1.** Multiple friction points discovered.

---

## Friction Point: Phantom "break" command

**Context**: During encounter-ice phase, I needed to break Palisade with Cleaver.

**Problem**: The system output suggests:
```
‚Üí Or break subs with: break "Palisade" <breaker>
```
But `break` is not a valid command. Running it returns `‚ùå Error: Unknown command: break`.

**Suggestion**: Either:
1. Implement the `break` command as suggested, OR
2. Change the hint to show the actual command: `use-ability "Cleaver" 2` (referencing the auto-pump-break ability)

**Impact**: High - confusing UX, wasted time figuring out real command

---

## Friction Point: `--full-break` flag not working

**Context**: Ran with `--full-break` expecting auto-break of ICE.

**Problem**: At 2¬¢ (before Pennyshaver payout), system said "Can't afford to break Palisade, letting subs fire". But then on subsequent runs with 8¬¢, `--full-break` still let subs fire without trying to break.

**Debug output showed**:
```
üîç DEBUG   [2] Match strength and fully break Palisade | playable=null dynamic=auto-pump-and-break
```
The `playable=null` might be the issue - ability wasn't detected as playable even with sufficient credits.

**Suggestion**:
1. Fix `--full-break` to actually attempt breaking when credits are available
2. Consider using Pennyshaver/hosted credits in the affordability calculation

**Impact**: High - core functionality broken

## Friction Point: Hand output lacks card abilities

**Context**: Needed to know Cleaver's pump/break costs.

**Problem**: `hand` command shows:
```

```
No ability text. I had to remember costs from game knowledge.

## Friction Point: Prompt shows wrong ICE strength

**Context**: Encountering Palisade protecting Server 1.

**Problem**: Prompt showed `Palisade (str 2, Barrier)` but Palisade protecting a remote should be str 5 (3 base + 2 from ability).

**Observation**: Cleaver's auto-break was 3¬¢ which suggests only 2 str differential (3‚Üí5 would need +2 pump at 2¬¢/str = 4¬¢ pump + break = more than 3¬¢). So maybe the +2 wasn't applying? Or the cost shown was actually for str 3 ICE?

**Suggestion**: Verify ICE strength calculation includes positional modifiers

**Impact**: Medium - incorrect info for tactical decisions

**Context**: Played Jailbreak event.

**Problem**: Output said `üÉè Played: Jailbreak` without indicating a prompt was pending. Had to run `prompt` separately to see the "Choose a server" choice.

   Choose a server: 0. HQ  1. R&D

## Friction Point: Status says "End of Turn" + "Waiting to start corp turn"

**Context**: After using all clicks, checking status.

**Problem**: Status showed both:
```
Clicks: 0 (End of Turn)
üí° Use 'end-turn' to finish your turn
```
AND
```
Status: üü¢ Waiting to start corp turn (use 'start-turn' command)
```

These are contradictory - am I waiting for the turn to end, or is Corp supposed to start?

**Suggestion**: Cleaner turn state messaging. If auto-end happened, don't show "use end-turn". If it didn't, don't show "waiting to start corp turn".

**Impact**: Low - confusing but doesn't block progress

---

## Friction Point: Start heuristic Corp is clunky

**Context**: Starting the heuristic Corp opponent.

**Problem**: Required running:
```bash
./dev/ai-eval.sh corp 7890 "(do (load-file \"./dev/src/clj/ai_heuristic_corp.clj\") (ai-heuristic-corp/start-autonomous!))"
```

This is error-prone and not discoverable.

**Suggestion**: Add `start-heuristic` command to send_command:
```bash
./dev/send_command corp start-heuristic
```

**Impact**: Low - one-time setup, but UX polish
